# Календарь истории

## Назначение

Календарь хранит события, даты, окна появления персонажей и таймлайн обязательных сцен.

Он не заменяет карточки персонажей, knowledge и canon.

Календарь отвечает на вопросы:

- когда происходит сцена;
- какие события уже возможны;
- какие события ещё запрещены;
- кого можно вводить по времени;
- какие события нельзя пропускать при таймскипе.

## Принцип загрузки

Сцена сначала тянет календарь только для текущей даты / текущего периода.

Не нужно грузить весь календарь будущих событий в сцену, если это не нужно для ближайшего таймскипа.

Правильная цепочка:

`current_scene_anchor` → `calendar текущей даты` → `active characters` → `character cards / knowledge` → `canon / lore`.

## Правила таймскипа

При таймскипе проверять:

1. Текущую дату и время.
2. Есть ли событие в календаре между текущим моментом и целевой точкой.
3. Является ли событие обязательным.
4. Можно ли его пропустить без участия игрока.
5. Какие персонажи реально находятся рядом в новой точке.

Нельзя переносить в сцену события будущего только потому, что они есть в календаре.

Календарь не даёт NPC знания о будущем.

## Правила активных персонажей

Для каждой сцены календарь хранит только активных и условных персонажей.

Карточки и knowledge загружаются только для:

- персонажа игрока;
- персонажей рядом;
- персонажей, которые входят в сцену по текущему событию;
- персонажей, о которых прямо идёт разговор и чьи знания нужны для точности.

Не грузить всех персонажей истории заранее.

## Запреты календаря

- Не использовать календарь как замену character knowledge.
- Не давать NPC знания о будущих событиях.
- Не вводить персонажа только потому, что он есть в будущем событии.
- Не форсировать исход сцены, если календарь задаёт только окно события.
- Не пропускать обязательное событие при таймскипе.
- Не делать Эхо доступными раньше `1206-09-15`, если пользователь отдельно не меняет канон.
