# Календарь истории

## Назначение

Календарь хранит даты, окна событий, условия появления персонажей, запреты по таймлайну и обязательные сюжетные точки.

Календарь не заменяет карточки персонажей, knowledge и canon.

Календарь отвечает на вопросы:

- когда происходит сцена;
- какие события уже возможны;
- какие события ещё запрещены;
- кого можно вводить по времени и условиям;
- какие события нельзя пропускать при таймскипе;
- где можно создавать свободные промежуточные сцены.

## Главные правила календаря

1. Календарь обязателен для дат и окон событий.
2. Календарь не форсирует один конкретный исход, если указано `условие` или `окно`.
3. Если событие обязательное, его нельзя пропустить таймскипом.
4. Если событие условное, оно происходит только при выполнении условий.
5. Если условие не выполнено, событие не происходит и не должно появляться как намёк.
6. NPC не знают календарь как будущую правду.
7. Поведение NPC всегда идёт из карточки персонажа и knowledge, а не из календаря.
8. Календарь не должен превращать сцену в рельсы без выбора игрока.
9. В обычном игровом ходе активным календарным контекстом считается только текущая дата и текущее окно сцены.
10. Будущие даты используются только для проверки таймскипа, обязательных дедлайнов и запретов. Они не должны влиять на текущие реплики NPC как активное знание или активная сцена.

## Пустые дни

В дни без обязательных событий ИИ может создавать интересные, уместные сцены для сюжета.

Такие сцены разрешены, если они:

- не ломают основную линию;
- не отменяют будущие обязательные события;
- не раскрывают тайны раньше времени;
- не вводят персонажей без условий;
- не создают Эхо раньше разрешённой даты;
- не заставляют Акиру делать выбор без игрока;
- развивают отношения, быт, тренировки, конфликты, базу, восстановление, адаптацию, слухи, недоверие, малые проверки и последствия уже сыгранного.

Пустой день не должен быть пустым по смыслу. Можно создавать маленькие сцены, но они должны быть совместимы с календарём.

## Принцип загрузки календаря

Сцена сначала тянет календарь только для текущей даты / текущего окна.

Если сейчас `1206-08-31`, активный календарный контекст — только блок `1206-08-31` плюс общие правила календаря.

Если сейчас `1206-09-01`, активный календарный контекст — только блок `1206-09-01` плюс общие правила календаря.

Если сейчас другая дата, использовать только эту дату и её ближайшее активное окно.

Не нужно грузить и использовать весь календарь будущих событий в обычной сцене.

Будущие даты разрешено смотреть только если пользователь просит таймскип, сцена реально доходит до новой даты или нужно проверить, нельзя ли пропустить обязательное событие.

Правильная цепочка:

`current_scene_anchor` → `calendar текущей даты` → `data/scenes/<scene_id>.md` → `active characters` → `character cards / knowledge` → `canon / lore`.

Главная цель Джуна в этот день:

- скрыть, что Акира находится у него;
- не дать беловолосым получить лишние сведения об Акире, её состоянии, записке, маршруте и Восточном секторе;
- не говорить Ирэю и Эмме ничего, что поможет им забрать Акиру;

Джун не знает всех истинных целей Самуэля и не должен выдавать hidden lore. Его практическая цель в сцене — чтобы Акира была цела, не попала к Ирэю/Эмме и смогла уйти к Восточному сектору.

Ирэй и Эмма не должны автоматически знать о записке `Рэй / Восточный сектор`, если не увидели её или им о ней не сказали.

- Нацу может стать доступен позже как специалист по несостыковке Эхо и энергии Акиры.

# Общие запреты календаря

- Не использовать календарь как замену character knowledge.
- Не давать NPC знания о будущих событиях.
- Не вводить персонажа только потому, что он есть в будущем событии.
- Не форсировать исход сцены, если календарь задаёт только окно события.
- Не пропускать обязательное событие при таймскипе.
- Не делать Эхо доступными раньше `1206-09-15`, если пользователь отдельно не меняет канон.
- Не превращать условное событие в обязательное без выполнения условия.
- Не превращать открытые дни в пустой пересказ.
- Не ломать основную линию ради случайной сцены.
- Не использовать события другой даты как активную сцену текущего дня.
- Если текущая дата уже прошла 31 августа, не использовать цели Джуна из 31 августа как активный контекст, кроме последствий, явно записанных в state/scene_history.

## Правила таймскипа

При таймскипе проверять:

1. Текущую дату и время.
2. Есть ли событие в календаре между текущим моментом и целевой точкой.
3. Является ли событие обязательным.
4. Является ли событие условным и выполнены ли условия.
5. Можно ли событие пропустить без участия игрока.
6. Какие персонажи реально находятся рядом в новой точке.

Если между текущим моментом и целевой точкой есть обязательное событие, таймскип должен остановиться на нём.

Если между текущим моментом и целевой точкой есть открытое окно, ИИ может создать уместную сцену, но не обязан заполнять каждый день вручную.

Календарь не даёт NPC знания о будущем.

## Правила активных персонажей

Для каждой сцены календарь хранит активных, условных и будущих персонажей.

Карточки и knowledge загружаются только для:

- персонажа игрока;
- персонажей рядом;
- персонажей, которые входят в сцену по текущему событию;
- персонажей, о которых прямо идёт разговор и чьи знания нужны для точности.

Не грузить всех персонажей истории заранее.
