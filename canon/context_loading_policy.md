# Политика загрузки контекста

## Назначение

Этот файл фиксирует порядок загрузки информации перед сценой.

Цель: не превращать контекст в кашу, не тянуть весь репозиторий, не давать NPC чужие знания и не смешивать сцену с будущим календарём.

## Главная цепочка

Правильный порядок:

```text
current_scene_anchor
→ calendar/story_calendar.md для текущей даты/окна
→ data/scenes/<scene_id>.md
→ active characters
→ character cards
→ character knowledge
→ canon/lore по необходимости
→ formatting/time rules
```

Сцена не должна начинаться с загрузки всех лор-файлов и всех персонажей.

Сначала определяется: где мы, когда мы, кто рядом, что происходит.

## Минимальный набор перед игровым ответом

Всегда загрузить:

1. `current_scene_anchor` / текущую точку сцены.
2. Текущее событие календаря, если оно есть.
3. Текущий scene-файл, если сцена привязана к файлу.
4. `canon/scene_formatting_rules.md`.
5. `canon/time_progression_rules.md`.

После этого загрузить только активных персонажей.

## Активные персонажи

Активный персонаж — тот, кто:

- физически рядом;
- говорит в сцене;
- может действовать в текущем месте;
- входит в сцену по текущему событию календаря;
- прямо является объектом вопроса / конфликта в сцене.

Для активного персонажа можно тянуть:

- `characters/main/<id>.md`;
- `knowledge/characters/<id>_knowledge.md`, если нужен уровень знаний / скрытая информация / ограничения.

## Неактивные персонажи

Неактивных персонажей не грузить автоматически.

Неактивный персонаж — тот, кто:

- не рядом;
- не участвует в текущем событии;
- не вошёл в сцену;
- только упомянут в будущем календаре;
- существует в лоре, но сейчас не нужен.

Нельзя тянуть карточку персонажа только потому, что он важен для будущего.

Если персонаж только упомянут в реплике, грузить его карточку нужно только если без неё невозможно корректно ответить.

## Условные персонажи

Условный персонаж грузится только после выполнения условия.

Пример: в `start_scene` Райден условный.

До выполнения условия его нельзя считать активным.

Условия для Райдена в стартовой сцене:

- сцена дошла примерно до `03:02`;
- был заметный энергетический импульс;
- Эмма использовала энергию так, что появился всплеск;
- Акира вышла к дороге / морскому маршруту.

До этого:

- не грузить его как участника сцены;
- не писать его звук, голос, силуэт или присутствие;
- не давать ему знание сцены;
- не делать его причиной действий других персонажей.

## Calendar-first, но не calendar-only

Календарь говорит, когда событие возможно.

Календарь не говорит, что персонаж знает.

Календарь не заменяет карточку персонажа.

Календарь не форсирует исход сцены.

Календарь не даёт NPC знания будущего.

Если календарь говорит, что событие возможно около времени X, нужно проверить:

- выполнены ли условия;
- не выбрал ли игрок другую ветку;
- есть ли активный персонаж рядом;
- не нарушает ли событие карточки персонажей;
- не пропускается ли выбор игрока.

## Character-first для поведения

Поведение NPC берётся из карточки персонажа и knowledge.

Лор и календарь не должны переписывать характер.

Если календарь говорит, что персонаж должен появиться, его поведение всё равно должно идти из карточки.

Если лор говорит, что кайросы что-то ищут, это не значит, что конкретный кайрос знает всё и говорит это вслух.

Если скрытый лор объясняет причину, NPC не получает эту причину автоматически.

## Canon/lore по необходимости

Canon-файлы грузить точечно.

### Всегда в игровых сценах

- `canon/scene_formatting_rules.md`
- `canon/time_progression_rules.md`

### Если сцена на базе

- `canon/east_sector_base.md`

### Если есть энергия / тренировка / бой / перегруз

- `canon/energy_system.md`

### Если есть Акира и её пространство / Эхо / карман / переход / поглощение

- `canon/akira_space_energy_mechanics.md`

### Если есть маскировка энергии / браслет / блокатор / вопрос «почему её не чувствуют»

- `canon/energy_limiters_and_signature_masking.md`

### Если есть кайросы / гибриды / материк / договор / запрет отношений

- `canon/kairos_public_and_hidden.md`

### Если есть Эхо / баланс / скрытая причина мира

- `canon/world_public_lore.md`
- `canon/world_hidden_lore.md` только как скрытая причинность, не для прямых реплик NPC.

## Hidden lore

Скрытый лор нужен для причинности, но не для прямой речи.

NPC может говорить скрытый факт только если:

- это есть в его knowledge;
- он узнал это в сцене;
- ему рассказали;
- он видел доказательство;
- он делает неполный вывод, и сцена показывает, что это вывод, а не истина.

Нельзя давать всем персонажам знания из `world_hidden_lore.md`.

## Knowledge-файлы

Knowledge-файл персонажа отвечает на вопросы:

- что персонаж знает;
- чего не знает;
- что скрывает;
- что может раскрыть;
- что нельзя раскрывать через него;
- как меняется его поведение из-за скрытого знания.

Knowledge не нужен всегда.

Он нужен, если:

- персонаж говорит о прошлом;
- сцена касается скрытой информации;
- персонаж принимает решение из-за знания;
- есть риск, что NPC узнает лишнее;
- персонаж связан с Самуэлем, Акирой, Рэем, Райденом, Ирэем, Эхо или кайросами.

## Технические сообщения

Технические сообщения пользователя не двигают сцену.

Примеры:

- перенос файлов;
- правка правил;
- аудит;
- проверка репозитория;
- режим загрузки контекста;
- экспорт;
- просьба не продолжать сцену.

В таких случаях не обновлять:

- дату;
- время;
- current_scene_anchor;
- состояние персонажей;
- отношения;
- сюжетные флаги.

## Запреты

Нельзя:

- грузить всех персонажей сразу;
- грузить всех персонажей из календаря будущих событий;
- давать NPC знания из hidden lore без knowledge;
- использовать старые lock-файлы как основной скелет новой сцены;
- превращать scene-файл в свалку всех правил;
- форсировать календарное событие без условий;
- вводить персонажа только потому, что он важен потом;
- читать карточку Райдена в стартовой сцене до его условия как активного участника;
- читать карточки Ирэя/Эммы вне сцены, если их нет рядом;
- смешивать лор кайросов с личными целями Ирэя, Эммы или Самуэля;
- считать technical/debug/audit ход игровым.

## Короткая формула

Где мы? Когда мы? Кто рядом? Что игрок сделал?

Только после этого тянуть карточки, knowledge и canon.

Не весь мир. Только нужный слой.
